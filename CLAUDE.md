# BAPLCP 排球報名系統

## 專案說明

BAPLCP 排球報名系統是一個提供排球活動報名、名單確認與報名狀態查詢的 Web 專案，主要使用於 LINE 生態內的活動參與情境。使用者可透過 LINE LIFF 開啟頁面，快速查看球局資訊、選擇報名項目、送出報名，並於活動頁面中確認自己的報名結果與候補狀態。

本專案採用純 Vue、JavaScript 開發，重點放在頁面載入速度、開發維護成本低、部署簡單，以及方便快速迭代 MVP 功能。網站以 GitHub Pages 作為靜態部署平台，並搭配 Supabase 作為後端資料儲存與資料存取服務，處理球局資料、報名紀錄、名單狀態與相關規則判斷。

此系統主要目標為：

- 提供使用者簡單直覺的排球活動報名流程

- 降低開團者整理名單與確認報名狀態的成本

- 讓參與者可以清楚知道自己是否報名成功、候補中或尚未補上

- 在不使用複雜前端框架的前提下，維持基本的互動體驗與可擴充性

---

## 使用情境

本系統主要使用於排球團、臨打團、固定球局或其他需要快速統計參加人數的活動情境。使用者通常會透過 LINE 群組、活動連結或指定入口進入 LIFF 頁面，在手機上完成以下操作：

- 查看目前可報名的球局資訊

- 選擇報名身份或人數

- 送出報名資料

- 查看名單與自己的報名狀態

- 確認是否已補上、候補中或未成功報名

對開團者而言，這個系統也能作為整理報名資料、顯示剩餘名額、控制報名規則與管理名單狀態的基礎工具。

---

## 專案目標

本專案以 MVP 為優先，目標不是一次完成所有複雜功能，而是先建立一套可用、可測試、可逐步擴充的報名流程。核心方向包括：

1. 建立穩定的報名主流程
   - 使用者可以順利完成報名操作

   - 系統可以正確寫入與顯示資料

2. 清楚呈現名單與狀態
   - 已報名、候補、已補上等資訊要容易理解

   - 降低使用者報名後的不確定感

3. 支援後續功能擴充
   - 預留未來加入更多球局規則、身份分類、截止邏輯、取消報名、候補遞補等機制的空間

4. 維持簡單技術棧
   - 避免過度工程化

   - 讓設計、前端與產品邏輯能快速同步與調整

---

## 技術架構

- 前端： Vue、JavaScript

- 後端：Supabase

- 部署：GitHub Pages

- 入口：LINE LIFF

### 前端

前端以 Vue、JavaScript 撰寫，負責頁面結構、視覺樣式、互動邏輯與資料顯示。可以覆用的部分盡量寫成 component、composable、util 等，以便維護與擴充

### 後端

Supabase 作為後端服務，主要負責：

- 儲存球局資料

- 儲存使用者報名資料

- 提供前端查詢與寫入所需的 API

- 支援後續名單狀態、截止時間、人數限制與權限規則

### 部署

專案部署於 GitHub Pages，適合此類靜態前端專案。此方式可降低部署門檻，並方便快速更新與版本管理。

### LINE LIFF

使用者主要從 LINE 環境進入頁面，因此在設計上需優先考慮手機瀏覽體驗、LINE 內開啟情境、操作流程簡潔，以及常見行動裝置尺寸下的可讀性與易用性。

---

## 核心功能

目前系統聚焦於以下核心功能：

- 球局資訊展示
  - 顯示活動名稱、日期、時間、地點、報名資訊與規則說明

- 報名操作
  - 使用者可選擇適合的報名方式與人數

  - 送出報名後寫入資料庫

- 名單與狀態顯示
  - 顯示目前名單、候補資訊與個人報名結果

  - 讓使用者知道自己是否成功補上

- 規則提示
  - 於報名前後提供必要說明

  - 減少「已送出但不一定成功」的誤解

## 設計與開發原則

為了讓專案在後續維護與迭代上更穩定，開發時請遵守以下原則：

- 每次只修改指定的檔案

- 改完後需明確說明修改了哪些地方

- 一律使用繁體中文回覆

- 優先維持既有結構與命名一致性

- 避免一次改動過多頁面或同時重構多個功能

- 若新增互動邏輯，需注意不影響既有報名流程

---

## 維護重點

由於本專案為純靜態前端搭配後端服務，維護時需特別注意以下事項：

- HTML 結構需保持可讀性，避免過度巢狀

- CSS 命名應盡量一致，避免樣式互相污染

- JavaScript 邏輯需區分頁面責任，避免全域變數混亂

- 能覆用的地方盡量寫成組件、composable、utils 等

- 與 Supabase 串接時，需清楚管理資料欄位與回傳資料格式

- 所有影響報名規則的邏輯都應有明確註解或文件說明

## 後續可擴充方向

- 報名截止時間設定

- 取消報名功能

- 管理者檢視與調整名單

- 與 LINE 使用者身份綁定

- 活動歷史紀錄查詢
